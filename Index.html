-- Chapter 6: Queries Within Queries – Subqueries
-- Example: Find all customers who have ordered a 'Laptop' (product_id 1001).
SELECT
    first_name,
    last_name
FROM CUSTOMERS
WHERE customer_id IN (SELECT customer_id

SELECT
    C.first_name,

Chapter 7: CTEs (WITH Clause)

Breaking down complex queries with Common Table Expressions.

Download: chapter_07_ctes.sql

Preview:

-- Chapter 7: Breaking Down Complexity – Common Table Expressions (WITH CTEs)
-- Example: Calculate the total cost for each line item in an order.
WITH OrderLineItemCosts AS (
    SELECT
        O.order_id,
        O.customer_id,
        O.quantity,
        P.price,
        (O.quantity * P.price) AS line_item_total
    FROM ORDERS AS O
    INNER JOIN PRODUCTS AS P ON O.product_id = P.product_id
)
SELECT *
FROM OrderLineItemCosts;

-- Exercise 7.1: Monthly Revenue Analysis (Solution)
-- Objective: Calculate the total_revenue for each month.
WITH OrderRevenue AS (
    SELECT
        order_id,
        (quantity * (SELECT price FROM PRODUCTS WHERE product_id = O.product_id)) AS revenue_per_line_item,
        order_date
    FROM ORDERS AS O
)
SELECT
    DATE_FORMAT(order_date, '%Y-%m') AS sale_month, -- For MySQL/PostgreSQL
    SUM(revenue_per_line_item) AS total_monthly_revenue
FROM OrderRevenue
GROUP BY sale_month
ORDER BY sale_month ASC;

Chapter 8: Window Functions

Performing calculations across related rows.

Download: chapter_08_window_functions.sql

Preview:

-- Chapter 8: Data Comparisons & Rankings – Window Functions
-- Example: For each order, calculate a running total of quantity ordered for that customer, ordered by date.
SELECT
    order_id,
    customer_id,
    order_date,
    quantity,
    SUM(quantity) OVER (PARTITION BY customer_id ORDER BY order_date ASC) AS running_customer_quantity
FROM ORDERS;

-- Exercise 8.1: Customer Lifetime Value (Simplified Running Total) (Solution)
-- Objective: For each order, list customer_id, order_date, product_name, quantity, and a running_total_quantity.
SELECT
    O.customer_id,
    O.order_date,
    P.product_name,
    O.quantity,
    SUM(O.quantity) OVER (PARTITION BY O.customer_id ORDER BY O.order_date ASC) AS running_total_quantity
FROM ORDERS AS O
INNER JOIN PRODUCTS AS P ON O.product_id = P.product_id;

Chapter 9: CASE Statements

Implementing IF/THEN/ELSE logic in your queries.

Download: chapter_09_case_statements.sql

Preview:

-- Chapter 9: Dynamic Logic – CASE Statements
-- Example: Categorize products by price segment.
SELECT
    product_name,
    price,
    CASE
        WHEN price < 100 THEN 'Budget'
        WHEN price BETWEEN 100 AND 500 THEN 'Mid-Range'
        WHEN price > 500 THEN 'Premium'
        ELSE 'Uncategorized'
    END AS price_segment
FROM PRODUCTS;

-- Exercise 9.1: Customer Loyalty Segment (Solution)
-- Objective: List customer_id, first_name, last_name, and their loyalty_segment.
SELECT
    C.customer_id,
    C.first_name,
    C.last_name,
    CASE
        WHEN COUNT(O.order_id) = 1 THEN 'New'
        WHEN COUNT(O.order_id) BETWEEN 2 AND 3 THEN 'Loyal'
        WHEN COUNT(O.order_id) >= 4 THEN 'VIP'
        ELSE 'No Orders'
    END AS loyalty_segment
FROM CUSTOMERS AS C
LEFT JOIN ORDERS AS O ON C.customer_id = O.customer_id
GROUP BY C.customer_id, C.first_name, C.last_name
ORDER BY C.customer_id;

-- Objective: Find category and total_quantity_sold for categories with total_quantity_sold > 3.
SELECT
    P.category,
    SUM(O.quantity) AS total_quantity_sold
FROM ORDERS AS O
INNER JOIN PRODUCTS AS P ON O.product_id = P.product_id
GROUP BY P.category
HAVING SUM(O.quantity) > 3
ORDER BY total_quantity_sold DESC;

5. Setup Guide: Get Started with Databites!

To start practicing SQL with the Databites database, follow these simple steps:

Step 1: Choose Your SQL Environment

  • Recommended for Beginners: SQLite

    SQLite is a file-based database, requiring minimal setup. Tools like DB Browser for SQLite provide a great graphical interface.

  • Popular Options: MySQL or PostgreSQL

    If you're comfortable with server-based databases, these are excellent choices. You'll need to install the database server and a client tool (e.g., MySQL Workbench, pgAdmin).

Step 2: Create the Databites Database Schema

  • Download the setup_databites_db.sql script from the "SQL Script Collection" above.
  • Open your chosen SQL client (e.g., DB Browser for SQLite, MySQL Workbench, pgAdmin).
  • Execute the `setup_databites_db.sql` script. This script will:
    • Drop existing tables (if any) to ensure a clean start.
    • Create the `CUSTOMERS`, `ORDERS`, and `PRODUCTS` tables with their respective columns and relationships.
    • Insert the sample data directly into these tables.
  • Verify: After execution, you should see the `customers`, `orders`, and `products` tables populated with data.

Step 3: Import Data (Optional, if not using `setup_databites_db.sql` for data)

If your environment prefers CSV imports, you can use the individual CSV files:

  • Download the customers.csv, orders.csv, and products.csv files.
  • Use your SQL client's import feature (e.g., "File -> Import -> Table from CSV file" in DB Browser for SQLite).
  • Ensure you map the columns correctly during import.

Step 4: Start Querying!

You're all set! Open the chapter-specific SQL scripts, copy the queries, and execute them in your database environment. Experiment, modify, and have fun mastering SQL!
